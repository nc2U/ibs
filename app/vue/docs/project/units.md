# 유닛 정보 설정

프로젝트의 동(건물) 정보를 등록하고, 해당 동에 속한 각 호수(유닛)를 일괄 또는 개별적으로 생성하고 관리합니다.

## 동(건물) 등록

프로젝트를 구성하는 각 동(건물)의 명칭을 등록합니다. 유닛을 등록하기 위해 반드시 선행되어야 하는 단계입니다.

- **위치**: `프로젝트 설정 > 유닛 정보 설정 > 동(건물) 등록`

#### 주요 입력 항목

| 라벨       | 필수입력 여부 | 비고                              |
|:---------|:-------:|:--------------------------------|
| 동(건물) 이름 |   필수    | 해당 건물을 식별할 명칭 입력 (예: 101, A동 등) |

> **주의**: 해당 동에 등록된 유닛(호수) 데이터가 있는 경우, 해당 유닛 데이터를 모두 삭제해야 동 정보를 삭제할 수 있습니다.

## 호(유닛) 등록

등록된 동(건물)별로 실제 분양/공급 대상인 호수(유닛) 정보를 생성합니다. 라인과 층 범위를 지정하여 일괄 등록하거나 개별 유닛의 상태(홀딩 등)를 수정할 수 있습니다.

- **위치**: `프로젝트 관리 > 유닛 정보 설정 > 호(유닛) 등록`

### 1. 호수(유닛) 일괄 등록

여러 층의 유닛을 라인별로 한 번에 생성하는 기능입니다.

#### 주요 입력 항목

| 라벨       | 필수입력 여부 | 비고                                |
|:---------|:-------:|:----------------------------------|
| 동(건물) 선택 |   필수    | 유닛을 생성할 대상 동 선택                   |
| 등록라인     |   필수    | 유닛을 생성할 라인 번호 (숫자 입력)             |
| 타입선택     |   필수    | 해당 라인/층에 적용할 유닛 타입(평형) 선택         |
| 시작층      |   필수    | 생성을 시작할 층 번호 (지하층은 음수 입력)         |
| 종료층      |   필수    | 생성을 종료할 층 범위 (시작층만 입력 시 1개 층만 생성) |

> **자동 생성 규칙**: 호수 명칭은 `층번호 + 라인번호` 조합으로 자동 생성됩니다. (예: 10층 1호 라인 -> 1001호)

### 2. 유닛 개별 관리 및 홀딩

생성된 유닛의 세부 정보를 수정하거나, 특정 사유로 분양을 제한(홀딩)할 때 사용합니다.

#### 주요 관리 항목

| 라벨       | 필수입력 여부 | 비고                                   |
|:---------|:-------:|:-------------------------------------|
| 타입       |   필수    | 해당 유닛의 타입 수정                         |
| 층범위타입    |   필수    | 분양가 산정 기준이 되는 층 범위 조건 연결             |
| 호수/라인/층수 |   필수    | 자동 생성된 유닛 명칭 및 위치 정보 수정              |
| 홀딩여부     |         | 체크 시 해당 유닛을 분양 제한 상태로 설정             |
| 홀딩사유     |         | 홀딩 설정 시 그 사유를 기재 (예: 회사보유분, 펜트하우스 등) |

> **참고**: 화면 왼쪽의 유닛 배치도를 통해 등록된 호수 현황을 시각적으로 확인할 수 있으며, 타입 정보 등록에서 설정한 색상에 따라 타입별로 구분되어 표시됩니다.
