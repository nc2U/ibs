# 설정

프로젝트의 전반적인 환경을 구성하고 관리하는 **설정** 페이지에 대한 사용자 가이드입니다.

## 1. 화면 개요
설정 화면은 프로젝트 관리자 및 권한이 있는 구성원이 프로젝트의 기본 정보, 참여 인원, 업무 프로세스 등을 정의하는 공간입니다. 탭 메뉴 방식을 통해 다양한 설정 항목을 체계적으로 관리할 수 있습니다.

## 2. 주요 설정 항목 안내

### 🏗️ 프로젝트 (기본 정보)
- **정보 수정**: 프로젝트 이름, 설명, 식별자(Slug) 등 기본 정보를 변경합니다.
- **모듈 활성화**: 업무(이슈), 달력, 게시판, 문서 등 해당 프로젝트에서 사용할 기능을 선택하여 활성화하거나 비활성화할 수 있습니다.

### 👥 구성원 관리
- **팀원 구성**: 프로젝트에 참여할 사용자를 추가하거나 제외합니다.
- **역할 및 권한**: 각 구성원에게 '관리자', '개발자', '보고자' 등의 역할을 부여하여 기능별 접근 권한을 세밀하게 제어합니다.

### 🛤️ 업무추적 및 범주
- **업무추적**: 해당 프로젝트에서 사용할 이슈 트래커(예: 결함, 지원, 기능 등)를 선택합니다.
- **업무범주**: 업무를 논리적으로 그룹화할 수 있는 범주(예: UI, 백엔드, 기획 등)를 생성하고 담당자를 지정하여 업무 배분을 자동화할 수 있습니다.

### 📅 단계 (마일스톤/버전)
- **일정 관리**: 프로젝트의 주요 마일스톤이나 출시 버전을 관리합니다.
- **진척도 확인**: 각 단계별로 할당된 업무의 진행률을 추적하여 전체 로드맵을 관리하는 기준이 됩니다.

### 📋 게시판 설정
- **소통 공간 정의**: 프로젝트 내에서 사용할 게시판의 이름과 설명을 설정합니다.
- **카테고리 구성**: 게시판 내에서 글을 분류할 카테고리를 추가하여 정보를 정돈합니다.

## 3. 주요 특징 및 이용 팁

### 🔐 권한에 따른 메뉴 노출
- 설정 메뉴는 사용자의 권한(`workManager` 또는 프로젝트별 부여 권한)에 따라 노출되는 항목이 달라집니다. 중요 설정(구성원 변경, 프로젝트 삭제 등)은 관리자 권한이 필요합니다.

### 🍪 마지막 탭 기억 기능
- 사용자가 마지막으로 머물렀던 설정 탭은 브라우저 쿠키를 통해 저장됩니다. 다음에 설정 화면에 다시 진입할 때 이전에 작업하던 탭이 자동으로 활성화되어 편리합니다.

### 💡 유연한 모듈 구성
- 프로젝트의 성격에 따라 필요한 모듈만 켜고 끌 수 있습니다. 예를 들어, 단순 공지용 프로젝트라면 '업무' 모듈을 끄고 '게시판'만 활성화하여 가볍게 운영할 수 있습니다.
