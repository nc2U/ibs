회의 관리 (Meetings) ​

회의 관리 모듈은 각 프로젝트(본사 관리 프로젝트 포함) 내에서 진행되는 다양한 회의 기록을 체계적으로 관리하고, 논의된 핵심 내용을 바탕으로 업무(Issue/Task)를 직접 등록하여 추적할 수 있도록 지원합니다.

1. 개요 및 접근 방법 ​

시스템 내의 모든 회의록은 예외 없이 특정 프로젝트에 반드시 소속되어야 합니다. 회사 본사 차원의 공통 업무 및 일반 회의 역시 별도의 '본사 관리' 프로젝트(또는 전사용 공통 프로젝트) 하위에 귀속되어 관리됩니다.

  • 회의 목록 접근:
    • 특정 프로젝트 선택 후 하위 메뉴의 [회의] 탭을 통해 접근하거나, 상단 메인 메뉴의 [회의]를 통해 전체 프로젝트의 회의 목록을 통합 조회 및 관리할 수 있습니다.
  • 회의 생성 권한:
    • '새 회의록' 생성 권한은 관리자(workManager) 또는 해당 프로젝트의 구성원에게만 부여됩니다. 권한이 없는 사용자는 생성 버튼이 노출되지 않습니다.
  • 프로젝트 상태에 따른 제약:
    • 선택한 프로젝트의 상태가 **'보관/종료' (상태 코드 9)**인 경우에는 해당 프로젝트 하위의 신규 회의록 생성이 제한되며, 기존 회의록 조회 및 편집만 가능합니다.

2. 회의록 목록 및 다차원 검색 ​

등록된 회의록 목록은 사용자가 전체적인 미팅 히스토리를 요약하여 신속하게 파악할 수 있도록 테이블 형태로 구성되어 있으며, 우측 사이드바를 통한 강력한 필터링 및 검색 기능을 제공합니다.

2.1 목록 정보 구성 ​

회의록 목록은 다음과 같은 열(Column)로 구성되어 있습니다.

  • 번호 (#): 회의 고유 식별 번호 (ID)
  • 프로젝트: 회의록이 소속된 프로젝트명 (상단 통합 [회의] 메뉴를 통한 조회 시 표시되며, 개별 프로젝트 내부 회의 목록에서는 목록 간결화를 위해 해당 열이 자동으로 숨겨집니다.)
  • 상태: 회의 진행 상태를 나타내는 상태 칩 (준비, 종료, 확정, 취소 등).
    • 확정 필요 경고: 상태가 '종료'('완료됨', 코드 2)인 회의록 중에서 회의일 이후 5일 이상 경과되었음에도 최종 확정되지 않은 경우, 목록에 확정 필요 배지가 추가 표시됩니다. 경과 기간에 따라 배지 색상이 차별화됩니다:
      • 6일 ~ 10일 경과: 주황색(Warning) 경고
      • 10일 초과: 빨간색(Danger) 경고
  • 카테고리: 프로젝트별로 등록된 회의의 종류나 분류 (예: 주간미팅, 정기회의, 기술보고 등)
  • 제목: 회의 주제 (취소 상태인 회의는 삭선(Strike-through) 처리되어 구분됩니다.)
  • 회의 일시: 분 단위까지 포맷팅된 실제 회의 일정
  • 작성자: 회의록을 최초 등록한 사용자
  • 참석: 총 참석자 수 (사내 사용자 및 기타 외부 참석자 수의 합계)
  • 등록일: 회의록 최초 생성 날짜
  • PDF 다운로드: 각 행 끝에 배치된 PDF 출력 아이콘(mdi-file-pdf-box)을 클릭하여 번거로운 화면 이동 없이 목록에서 바로 PDF 문서를 다운로드할 수 있습니다.

2.2 우측 사이드바 검색 및 필터링 ​

오른쪽 사이드바 영역의 필터를 통해 대량의 회의록 중에서 목적에 맞는 정보를 빠르게 추려낼 수 있습니다.

  • 회의 일시: 특정 일자별 회의 일정 필터링 (달력 위젯 제공)
  • 회의 상태: 준비중(1), 완료됨(2), 취소됨(3) 등의 진행 상태 필터링
  • 카테고리: 선택한 프로젝트에 등록된 미팅 분류 목록 필터링
  • 검색어: 검색창에 입력한 키워드를 바탕으로 회의 제목 및 본문 내용 전체를 포함 검색합니다.
  • 적용: '적용' 버튼 클릭 시 필터 조건이 반영되어 목록이 새롭게 갱신되며, 검색 시 페이징 번호는 1페이지로 자동 초기화됩니다.

3. 회의록 작성 및 수정 (Form) ​

새 회의록을 작성([새 회의록])하거나 기존 정보를 수정([수정])할 수 있는 폼 인터페이스입니다. 화면은 좌측의 상세 정보 입력창과 우측의 메타 정보 설정창으로 나누어져 작업 편의성을 극대화했습니다.

3.1 좌측 상세 정보 입력창 ​

  • 회의 제목: 회의 명칭을 입력합니다. (필수 입력 항목, 미입력 시 경고 노출)
  • 회의 의제: 회의 전 사전에 공유하거나 논의할 핵심 어젠다를 입력하는 멀티라인 텍스트 입력창입니다.
  • 회의 내용 (Markdown 지원): 실제 회의가 진행되는 과정을 마크다운 에디터(MdEditor)를 사용하여 실시간 서식 편집(볼드, 헤더, 리스트 등)과 함께 상세하게 기록합니다.
  • 주요 결정 사항: 회의 과정에서 최종 확정되거나 합의된 중요 결정 내용을 기술하는 전용 입력 공간입니다. (Markdown 포맷팅이 적용되어 상세 보기 시 강조됩니다.)
  • 후속 조치 사항: 회의 이후 누가, 언제까지, 무엇을 완수할 것인지(To-Do)에 대한 조치 사항을 작성합니다. (상세 보기 시 경고/조치 성격으로 강조 표기됩니다.)
  • 파일 첨부:
    • 다중 파일 업로드를 지원합니다.
    • 첨부하는 개별 파일마다 설명란(부가적인 설명)을 각각 입력할 수 있습니다.
    • 삭제가 필요한 첨부 파일은 우측의 휴지통 아이콘을 클릭하여 손쉽게 제외할 수 있습니다.

3.2 우측 메타 정보 설정창 ​

  • 프로젝트: 회의록이 귀속될 프로젝트를 지정합니다. (필수 사항)
  • 회의 일시: 회의 일자와 시간을 선택할 수 있는 캘린더 및 시간 위젯(DateTimePicker)을 제공합니다.
  • 카테고리: 회의 카테고리를 지정합니다.
    • 실시간 추가 지원: 만약 목록에 원하는 카테고리가 없다면, 선택 상자 우측의 + (추가) 버튼을 눌러 나타나는 모달 창을 통해 즉석에서 새로운 카테고리(카테고리명, 지정 색상, 정렬 순서)를 생성하여 곧바로 지정할 수 있습니다. 생성된 카테고리는 해당 프로젝트에 귀속됩니다.
  • 상태: 회의록 진행 수준을 설정합니다. 준비(1), 종료(2), 확정(3), 취소(4) 중 알맞은 단계를 선택할 수 있습니다.
    • '확정'('3') 상태 선택 제한: 회의록의 내용을 공식 승인하는 '확정' 단계는 관리자(workManager)만 선택할 수 있습니다. 일반 사용자가 작성/수정 시에는 목록에서 '확정' 옵션이 노출되지 않습니다. (단, 이미 '확정' 상태인 회의록을 수정할 경우 예외적으로 옵션이 유지됩니다.)
  • 참석자: 사내 시스템 사용자들을 대상으로 이름을 검색하여 멀티 초이스 칩(Chips) 형태로 다중 지정합니다. 불필요한 인원은 칩의 삭제 아이콘(x)을 눌러 빠르게 제거 가능합니다.
  • 기타참석: 외부 거래처나 시스템 계정이 없는 비회원 참석자를 자유롭게 텍스트로 기입합니다.

3.3 관련 업무 등록의 제약 사항 ​

  • 주의: 신규 회의록을 최초 작성(등록)하는 시점에는 회의 의존적인 업무를 추가할 수 없습니다. 회의록의 식별자(PK)가 생성되지 않았기 때문입니다.
  • 회의 내용과 유기적으로 결합될 업무(Issue)를 신규 생성하거나 연결하려면, 회의록 작성을 완료하여 저장한 이후, '수정' 또는 '상세 보기' 화면에서 진행해야 합니다.

4. 회의 상세 보기 및 관련 업무(Issue) 스마트 연동 ​

회의록 등록을 마치면 작성된 상세 페이지가 노출되며, 기록 공유뿐만 아니라 실제 실무 업무(Issue/Task)를 동적으로 추적 및 생성할 수 있는 연동 제어판 역할을 제공합니다.

4.1 회의록 수정 및 삭제 권한 ​

  • 수정 권한 (수정 버튼):
    • 미확정 회의록(준비/종료 상태): 관리자, 회의록 생성자, 혹은 회의 참석자인 경우 수정 버튼이 노출됩니다.
    • 확정 회의록: 관리자(workManager)만 수정할 수 있습니다.
  • 삭제 권한 (삭제 버튼):
    • 미확정 회의록(준비/종료 상태): 관리자 혹은 회의록 생성자인 경우 삭제 버튼이 노출됩니다.
    • 확정 회의록: 관리자(workManager)만 삭제할 수 있습니다.
  • 권한이 없는 사용자에게는 해당 버튼이 자동으로 숨겨집니다.

4.2 시각적인 레이아웃 및 마크다운 렌더링 ​

  • 메타 데이터 및 작성 기록: 작성자 정보와 최초 등록 경과 시간(툴팁 시 절대 시간 표시), 최종 수정자 정보 및 최종 수정 시간 정보가 세밀하게 표시됩니다.
  • 마크다운 렌더링: '회의 의제', '회의 내용'이 깔끔한 마크다운 서식으로 실시간 변환되어 가독성이 뛰어납니다.
  • 전용 콜아웃(Callout) 블록:
    • 주요 결정 사항은 녹색 성공 콜아웃(CCallout color="success") 블록으로 감싸져 한눈에 확인할 수 있습니다.
    • 후속 조치 사항은 주황색 경고 콜아웃(CCallout color="warning") 블록으로 자동 스타일링되어 담당자가 진행 사항을 직관적으로 리마인드할 수 있도록 유도합니다.

4.3 업무(Issue/Task) 연동 상태 및 실시간 트래킹 ​

  • 완료율 프로그레스 바: 회의록에 등록되어 있는 관련 업무 목록을 실시간 분석하여 **완료 건수 / 전체 건수 (진행률 %)**로 가시화합니다. 하단에 진행 상태에 맞춰 색상이 변하는 시각적 프로그레스 바(진행률 100% 도달 시 초록색 success 전환, 미완료 시 파란색 primary 유지)를 보여줍니다.
  • 연결 업무 테이블: 해당 미팅과 연결된 모든 업무가 일련번호, 제목, 상태 칩(완료 여부에 따른 색상 대응), 그리고 담당자 정보와 함께 목록 형태로 투명하게 조회됩니다. 업무 제목 클릭 시 해당 업무의 상세 보기로 즉시 페이지 이동합니다.
  • 실시간 관련 업무 추가 및 편집:
    • 상세 보기 페이지 및 수정 화면의 관련 업무 영역에서 [관련 업무 추가] 버튼을 통해 현재 회의에 자동으로 연결되는 신규 업무를 모달 창(IssueForm) 안에서 다이렉트로 작성할 수 있습니다.
    • 회의록 수정 화면에서는 기존 연결된 업무들의 상세 내용을 확인하고 우측의 연필(Edit) 아이콘을 눌러 실시간으로 업무 명세나 담당자, 상태를 모달에서 변경 및 저장하여 즉시 반영하는 유연함을 제공합니다.

4.4 첨부 파일 및 상단 액션 바 ​

  • 첨부 파일 뷰어: 첨부된 모든 파일들이 소유자 정보와 함께 하단 첨부 파일 구역에 파일 카드로 깔끔하게 표시됩니다.
  • 회의록 PDF 출력: 우측 상단의 [PDF 출력] 버튼을 클릭하여 본 회의록 내용 전체를 아름답게 구성된 문서 레이아웃의 PDF 파일로 영구 내보내기 할 수 있습니다.

5. 활동 기록(Activity Log) 연동 ​

회의록의 신규 생성, 본문 수정 등의 모든 히스토리는 해당 프로젝트의 [실행 기록] 피드(Activity Feed)에 시스템 로그로서 즉각 기록됩니다. 타 팀원들은 실시간으로 생성되는 실행 기록 피드에서 미팅 내용 요약을 한눈에 모니터링하고 링크를 통해 신속하게 세부 회의록으로 진입하여 회의 결과를 동기화할 수 있습니다.

6. 캘린더 연동 ​

등록된 회의록은 '업무 달력' 기능과 통합되어 시각화됩니다.

  • 캘린더 확인: 전역 달력 또는 프로젝트별 달력 화면에서 등록된 회의 일정을 확인할 수 있습니다.
  • 회의 구분: 회의 일정은 업무(Issue)와 명확히 구분되도록 **보라색 배경([회의] 접두어)**으로 표시됩니다.
  • 상세 이동: 캘린더상의 회의 일정을 클릭하면 해당 회의의 상세 페이지로 즉시 이동하여 내용을 확인할 수 있습니다.
